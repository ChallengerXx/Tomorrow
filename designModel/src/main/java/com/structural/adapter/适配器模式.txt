1.优点：
    1.1 可以让两个不相关的类一起运行
    1.2 提高了类的复用，如果想使用现有的类，而这个类又不符合现有系统的需要，可以通过适配器模式让这些功能得到复用
    1.3 增加了类的透明度，客户端只关注结果
    1.4 使用适配器的时候，可以调用自己开发的功能，从而自然的扩展系统功能
    1.5 总的来说，适配器模式属于补偿模式，主要在系统后期扩展，修改时使用
2.缺点：
    2.1 过度的使用适配器模式，会导致系统凌乱，追溯困难（内部转发导致，调用A适配成B）
3.适用场景：
    3.1 系统需要适用一些现有的类，但是这些类的接口或者方法名不符合系统的需要，甚至没有这些类的源代码
    3.2 想要创建一个可以重复适用的类，用于同一些没有太大关联的类，或者是之后可能引进的一些类一同工作
